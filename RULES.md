# Commit Message Rules

## Format
<type>(<scope>): <summary>

## Types
| type     | when to use                        |
|----------|------------------------------------|
| feat     | new function / feature             |
| fix      | bug fix                            |
| math     | equation / algorithm correction    |
| refactor | code restructure, no behavior change|
| docs     | comments, docstrings, markdown     |
| test     | add or fix tests                   |
| sim      | simulation scripts                 |
| perf     | performance improvement            |
| chore    | config, dependencies               |
| wip      | work in progress                   |

## Scopes
lc | ks | cart | cr3bp | transform | eom

## Rules
- English, imperative mood ("add", not "added")
- Summary ≤ 50 characters, no period
- Body: explain *why*, not *what*
- Reference paper/equation in footer if relevant
